# Pourquoi ce kit

Ce kit est aligné avec les docs Goose :
- les sessions gardent le contexte et l'historique,
- les projects permettent de reprendre un codebase rapidement,
- les subagents gardent la conversation principale propre,
- les recipes permettent de rendre les workflows répétables et validables,
- le pattern RPI (Research → Plan → Implement) est recommandé pour les changements complexes,
- `goose run` / headless est utile pour exécuter des tâches automatisées,
- le Playwright CLI skill est recommandé pour l'automatisation navigateur de façon plus économe en tokens.
